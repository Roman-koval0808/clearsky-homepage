ClearSky Google Review Test Suite - Full 12-Step Pipeline
Target URL: http://localhost:5173/api/signals/google/review

These commands test the complete journey from Raw Data (Step 1) to Signal Generation (Step 12).
NOTE: Change "review_id" for each run to avoid "Duplicate Blocked" (Step 7).

--------------------------------------------------------------------------------
SCENARIO 1: The Raving Fan (Momentum + Testimonial)
Expected Signals: positive_review_received, service_quality_praise_detected, 
                  testimonial_candidate_detected, service_proof_point_detected
--------------------------------------------------------------------------------
curl -X POST http://localhost:5174/api/signals/google/review \
     -H "Content-Type: application/json" \
     -d '{
      "provider": "google_business_profile",
      "review_id": "sig_test_fan_'$(date +%s)'",
      "location_id": "gbp_location_1199",
      "author_name": "John Doe",
      "rating": 5,
      "comment": "Absolutely incredible service! The team replaced my entire roof in two days. Cleaned up everything. Best contractor in Timmins.",
      "created_time": "'$(date -u +"%Y-%m-%dT%H:%M:%SZ")'"
     }'

HOW TO VERIFY SIGNALS:
--------------------------------------------------------------------------------
1. Check the server console logs for Step 10, 11, and 12.
2. Run this SQL in your database to see the signals created for the latest event:

SELECT e.event_id, e.review_rating_numeric, s.name, s.bucket 
FROM events e
JOIN signals s ON e.id = s.event_id
ORDER BY e.created_at DESC
LIMIT 5;
